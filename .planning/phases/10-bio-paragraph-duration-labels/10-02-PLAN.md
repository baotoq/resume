<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| YAML data -> WorkExperience render | startDate/endDate come from developer-controlled resume.md; not user input |
| computeDuration output -> JSX | Return value is a plain string inserted as React text child — auto-escaped |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-10-03 | Tampering | WorkExperience.tsx | accept | startDate/endDate are git-committed YAML values, not user-supplied. computeDuration output is a plain string — React text child auto-escapes any special chars. No XSS risk. |
| T-10-04 | Information Disclosure | Server Component render | accept | Duration is a public label derived from public resume data. No PII, no secrets in duration strings. |
| T-10-05 | Denial of Service | computeDuration with malformed date | accept | Malformed "YYYY-MM" strings produce NaN arithmetic, yielding totalMonths = NaN. `NaN < 1` is false in JS, so NaN falls through to `Math.floor(NaN/12)` = NaN, `NaN % 12` = NaN, returns `NaN yrs NaN mos` — ugly but not a crash. Risk accepted because input is developer-controlled YAML, not user input. |
</threat_model>
