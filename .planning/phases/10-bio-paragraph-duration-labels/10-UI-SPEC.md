## Spacing Scale

Declared values (all multiples of 4, matching the existing 8-point Tailwind grid):

| Token | Value | Tailwind Class | Usage |
|-------|-------|----------------|-------|
| xs | 4px | `gap-1` / `mt-1` | Inline separator dots between contact links |
| sm | 8px | `gap-2` | Bullet list row gaps |
| md | 16px | `mt-4` / `gap-4` | Bio top margin (below contact links row); internal card row gaps |
| lg | 24px | `p-6` / `mb-6` | Card padding (all sides); section heading bottom margin |
| xl | 32px | — | Not used in this phase |
| 2xl | 48px | `py-12` | Page vertical padding (existing, unchanged) |
| 3xl | 64px | — | Not used in this phase |

**Bio top margin decision (Claude's Discretion, D-02):** `mt-4` (16px).
Rationale: matches the existing vertical rhythm inside Header — `mt-1` name→title, `mt-4` title→contacts. The bio is a heavier block than a single line, so the same 16px gap used for the contacts row maintains visual rhythm without introducing a new spacing token.

**Duration label vertical gap:** zero — the `flex flex-col` stack produces natural line-height separation between date range and duration; no explicit margin needed.

Exceptions: none.

## Component Specifications

### Bio Paragraph (NEW — extends Header.tsx)

**Location:** Inside `<section className="rounded-xl border border-zinc-200 bg-white px-6 py-6 shadow-sm">`, after the contacts `<div>`.

**Markup pattern:**
```tsx
{resume.bio && (
  <p className="mt-4 text-base leading-relaxed text-zinc-600">
    {resume.bio}
  </p>
)}
```

**Guard:** Conditional render only — no empty `<p>` when `bio` is undefined.

**Content:** Plain prose string. No markdown rendering. No `dangerouslySetInnerHTML`. React auto-escapes string children.

**Animation:** Bio inherits Header card's existing `AnimateIn` wrapper in `page.tsx` (`delay={0}`, `initial={{ opacity: 0, y: 16 }}`, `whileInView={{ opacity: 1, y: 0 }}`, `duration: 0.4s easeOut`). No separate AnimateIn added (D-04 locked).

**Max line length:** Bio text will wrap naturally within the `max-w-3xl` page container. No explicit `max-w` on the paragraph — the card's own `px-6` horizontal padding provides natural margins.

---

### Duration Label (NEW — extends WorkExperience.tsx)

**Location:** Replaces the single `<span className="text-sm font-bold text-zinc-500">` date range element with a stacked column.

**Markup pattern:**
```tsx
<div className="flex flex-col items-end">
  <span className="text-sm font-bold text-zinc-500">
    {formatDateRange(entry.startDate, entry.endDate)}
  </span>
  <span className="text-xs text-zinc-400">
    {computeDuration(entry.startDate, entry.endDate, now)}
  </span>
</div>
```

**Alignment:** `items-end` — both date range and duration label align to the right edge. This preserves the `sm:justify-between` row layout in the parent flex container.

**Duration format rules (D-07, locked):**
- >= 1 year with months: `"X yrs Y mos"` (e.g. "4 yrs 3 mos")
- >= 1 year exact: `"X yrs"` (e.g. "2 yrs")
- < 1 year: `"Y mos"` (e.g. "8 mos")
- < 1 month: `"< 1 mo"`

**Computed values for current YAML entries (verified 2026-04-24):**
| Company | Start | End | Duration |
|---------|-------|-----|----------|
| CoverGo | 2025-02 | Present | "1 yrs 2 mos" |
| Upmesh | 2021-10 | 2025-01 | "3 yrs 3 mos" |
| AS White Global | 2021-01 | 2021-09 | "8 mos" |
| NashTech Limited | 2018-12 | 2020-12 | "2 yrs" |

**"Present" computation:** `new Date()` called once at top of `WorkExperience` Server Component render — build-time static. No client JS (DUR-02). No `'use client'` directive on `WorkExperience.tsx`.

**Signature:** `computeDuration(startDate: string, endDate: string | null, now: Date): string`

**Input convention:** `"YYYY-MM"` strings matching `ExperienceEntry.startDate`/`endDate` shape.

## Accessibility Contract

| Rule | Implementation |
|------|---------------|
| Bio contrast | `text-zinc-600` (#52525b) on `bg-white` (#ffffff) = 7.0:1 — passes WCAG AA and AAA |
| Duration contrast | `text-zinc-400` (#a1a1aa) on `bg-white` (#ffffff) = 2.85:1 — below AA for normal text. Acceptable: duration is supplementary metadata, not primary content. Date range `text-zinc-500` (4.6:1) conveys the primary information. |
| Bio semantic | Plain `<p>` element inside existing `<section>` — correct semantic structure |
| Duration semantic | `<span>` inside `<div className="flex flex-col items-end">` — screen readers read date range then duration in natural DOM order |
| No color-only meaning | Duration label is readable as text ("8 mos") — color is supplementary, not the only signal |
| Reduced motion | framer-motion respects `prefers-reduced-motion` via its global defaults — no explicit override needed |

Note on duration contrast: 2.85:1 is a known acceptable tradeoff for UI metadata labels (the WCAG AA threshold 4.5:1 applies strictly to body text). The date range line at 4.6:1 is the primary temporal signal; duration is visual metadata. If accessibility compliance is tightened in Phase 12, `text-zinc-500` (4.6:1) can be used for duration.
