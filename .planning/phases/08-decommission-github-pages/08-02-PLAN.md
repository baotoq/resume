<task type="checkpoint:human-action" gate="blocking">
  <name>Task 1: Disable GitHub Pages in repository settings (per D-01)</name>
  <read_first></read_first>
  <what-to-do>
Disable GitHub Pages for this repository. This cannot be done via CLI — it requires the GitHub web UI.

Steps:
1. Go to https://github.com/{your-username}/{your-repo}/settings/pages
   (Replace {your-username}/{your-repo} with the actual repo path — check `git remote get-url origin` if unsure)
2. Under "Build and deployment", find the "Source" dropdown
3. Change Source from its current value to "None"
4. Click "Save"
5. GitHub will show a confirmation that Pages has been disabled

After saving, GitHub Pages will be deprovisioned. The old `*.github.io` URL will stop serving your site (may take a few minutes to propagate).
  </what-to-do>
  <resume-signal>Type "disabled" once you have saved the Pages source to None in GitHub settings</resume-signal>
</task>

<task type="auto">
  <name>Task 2: Verify old GitHub Pages URL returns 404 (per D-02)</name>
  <read_first></read_first>
  <files></files>
  <action>
Determine the old GitHub Pages URL from the git remote, then curl it to confirm 404.

Step 1 — Get the repo remote URL:
```bash
git remote get-url origin
```

Step 2 — Derive the Pages URL. GitHub Pages URLs follow the pattern:
- For a repo named `resume` owned by user `baotoquoc-covergo` → `https://baotoquoc-covergo.github.io/resume/`
- For a user/org site named `{username}.github.io` → `https://{username}.github.io/`

Step 3 — Curl the Pages URL and check the HTTP status:
```bash
curl -o /dev/null -s -w "%{http_code}" https://{pages-url}/
```

Expected result: `404` (or `301`/`302` redirect to a 404, or connection refused — all indicate Pages is disabled).

If the result is `200`, GitHub Pages propagation may still be in progress. Wait 2-3 minutes and retry. If still 200 after 5 minutes, flag for the human — Pages may not have been disabled correctly.

Step 4 — Confirm Vercel deployment is still live:
```bash
curl -o /dev/null -s -w "%{http_code}" https://{vercel-url}/
```
Expected result: `200`.

Note: The Vercel URL can be found in `.planning/phases/07-vercel-setup-config-migration/07-01-SUMMARY.md` or by running `cat .planning/phases/07-vercel-setup-config-migration/07-01-SUMMARY.md`.
  </action>
