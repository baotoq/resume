## Component Contract: HighlightedBullet.tsx

**Location:** `src/components/HighlightedBullet.tsx`
**Type:** React Server Component (no `'use client'` directive)
**Pattern:** Mirrors `TechStackIcons.tsx` — single focused prop, no state, no effects.

### Props

| Prop | Type | Required | Description |
|------|------|----------|-------------|
| `children` | `string` | yes | Raw bullet text from resume.md, may contain `**bold**` or `*italic*` markdown |

### Inline Element Rendering Contract

| Input pattern | Output element | Tailwind classes applied | Semantic meaning |
|---------------|---------------|--------------------------|-----------------|
| `**text**` | `<span>` | `text-indigo-600` | Keyword accent — color only, weight unchanged |
| `*text*` | `<span>` | `italic` | Italic emphasis |
| Plain text segments | React text node | none | Rendered as-is |
| Any other syntax (`[url]`, `#`, etc.) | React text node | none | Passed through as literal characters — no transformation |

### Parsing Order

Bold is parsed before italic to correctly handle `***triple***` edge cases:
1. Split on `**...**` — extract bold segments
2. Within each non-bold segment, split on `*...*` — extract italic segments
3. All remaining text is plain

`***bold italic***` (triple asterisk): pass through as literal text — the simpler safe path (per D-04 discretion in CONTEXT.md).

### Integration Point

`WorkExperience.tsx` line 68 — current code:
```
{bullet}
```
Replace with:
```
<HighlightedBullet>{bullet}</HighlightedBullet>
```

The `<li>` element retains all existing classes (`text-base leading-relaxed text-zinc-700 pl-4 relative before:...`). `HighlightedBullet` renders inline spans inside it — no wrapping block element.

## States

| State | Visual | Notes |
|-------|--------|-------|
| No markup (plain bullet) | Identical to current — zinc-700, no spans | KW-03 requirement — zero regression |
| Bold keyword | Inline `<span class="text-indigo-600">` wrapping keyword text | Color only, weight 400, size 16px |
| Italic keyword | Inline `<span class="italic">` wrapping italic text | Style only, no color change |
| Bold + italic overlap (`***`) | Literal asterisks rendered as text | Safe fallback, no compound span |

No loading states, no error states, no empty states — this is a pure synchronous render of static string data. No async data fetching.

## Accessibility Contract

| Requirement | Implementation |
|-------------|---------------|
| No `<b>` or `<strong>` for visual-only styling | Use `<span class="text-indigo-600">` — color is the sole signal, no semantic weight |
| No `<em>` or `<i>` for decorative italic | Use `<span class="italic">` — Tailwind utility, no semantic emphasis |
| Screen reader behavior | Inline spans with no `aria-*` attributes — content reads as continuous text, which is correct |
| Color contrast | indigo-600 (`#4f46e5`) on white card background — WCAG AA passes for normal text (contrast ratio ~5.9:1) |
| Color not sole differentiator of meaning | Bold keywords are identified by position and content in resume.md; color is decorative aid only |
