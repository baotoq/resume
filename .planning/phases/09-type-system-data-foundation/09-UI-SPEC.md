## Spacing Scale

Declared values (multiples of 4, derived from existing component usage):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, inline separators |
| sm | 8px | Compact element spacing, bullet list gaps (`gap-2`) |
| md | 16px | Default element spacing (`gap-4`, `mb-4`) |
| lg | 24px | Card padding, section-internal spacing (`p-6`, `mb-6`) |
| xl | 32px | Layout column gaps |
| 2xl | 48px | Major section breaks |
| 3xl | 64px | Page-level top/bottom padding |

Source: Derived from `WorkExperience.tsx` and `Header.tsx` class usage — `p-6`, `gap-4`, `gap-6`, `mb-6`, `mt-4`, `gap-2`.

Exceptions: Timeline rail uses non-standard offsets (`-left-5.5`, `-left-6.5`) for dot alignment — these are implementation details, not spacing tokens. Do not attempt to normalize to the 4-point scale.

**Phase 9 constraint for downstream phases:** Bio and education components MUST use `p-6` card padding and section-level gap of `mb-6` or `gap-6` to remain consistent with `WorkExperience`. Phase 12 will enforce `gap-10`/`gap-12` between sections (TYP-02) — do not preempt this.

## Typography

Derived from live component inspection (`Header.tsx`, `WorkExperience.tsx`):

| Role | Size | Weight | Line Height | Tailwind Class | Usage |
|------|------|--------|-------------|----------------|-------|
| Display | 28px | 600 (semibold) | 1.1 | `text-[28px] font-semibold leading-[1.1]` | Name in Header |
| Heading | 20px | 600 (semibold) | 1.2 | `text-xl font-semibold leading-[1.2]` | Section headings (e.g. "Work Experience") |
| Subheading | 18px | 700 (bold) | default | `text-lg font-bold` | Role titles within cards |
| Body | 16px | 400 (regular) | relaxed (~1.625) | `text-base leading-relaxed` | Bullet text, body prose |
| Secondary | 14px | 700 (bold) | default | `text-sm font-bold` | Date ranges, meta labels |

Source: `Header.tsx` lines 18-19 (display/heading), `WorkExperience.tsx` lines 59-71 (subheading/secondary), line 79 (body).

**Phase 9 contract for downstream Bio component (Phase 10):**
- Bio paragraph: `text-base leading-relaxed text-zinc-700` — matches bullet body style.
- No heading needed for the bio block itself; it sits directly below the Header section.

**Phase 9 contract for downstream Education component (Phase 11):**
- Section heading: `text-xl font-semibold leading-[1.2] text-zinc-900` — matches "Work Experience" heading exactly.
- Institution name: `text-lg font-bold text-zinc-900` — parallel to role title in work cards.
- Degree label: `text-base font-semibold text-zinc-700` — one weight step below institution.
- Date range: `text-sm font-bold text-zinc-500` — matches work experience date style.

**Phase 12 will override font family to Inter.** Downstream phases must NOT hardcode Geist — rely on `var(--font-sans)` via `font-family: var(--font-sans)` inherited from `body` in `globals.css`.

## Data Contract (Phase 9 Primary Output)

This section captures the TypeScript and YAML contracts that Phase 9 establishes. Downstream phases are bound by these contracts.

### TypeScript Interface: `EducationEntry`

```typescript
export interface EducationEntry {
  degree: string;
  institution: string;
  startDate: string;    // "YYYY-MM" format
  endDate: string | null; // null = currently enrolled; "YYYY-MM" = graduated
  logo_url?: string;
  link?: string;
}
```

Source: CONTEXT.md D-04 through D-08; RESEARCH.md Pattern 1.

### TypeScript Interface: `ResumeData` additions

```typescript
bio?: string;               // plain prose, no markdown
education?: EducationEntry[];
```

Source: CONTEXT.md D-01, D-09; RESEARCH.md Pattern 2.

### YAML Data Shape

```yaml
bio: "<verbatim text from D-02>"
education:
  - degree: "Bachelor of Computer Science"
    institution: "Ton Duc Thang University"
    startDate: "2014-09"
    endDate: "2018-06"
```

Source: CONTEXT.md D-10; RESEARCH.md Pattern 3.

### Constraints downstream phases must not violate

1. `bio` is plain prose — no markdown rendering (explicitly out of scope, REQUIREMENTS.md "Out of Scope" table).
2. `education` is an array — Phase 11 must guard for empty array and `undefined` (field is optional).
3. `endDate: string | null` — not `endDate?: string`. The `null` value means "currently enrolled". Phase 11 must render "Present" when `endDate === null`.
4. `logo_url` and `link` are absent from the initial YAML entry — Phase 11 must not crash when these optional fields are absent.
5. `startDate`/`endDate` keys are camelCase in YAML — no transformation. Read directly as `entry.startDate`, `entry.endDate`.
