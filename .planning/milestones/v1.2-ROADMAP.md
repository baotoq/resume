# Milestone v1.2: Tech Stack Icons + Keyword Highlights

**Status:** ✅ SHIPPED 2026-04-14
**Phases:** 5-6
**Total Plans:** 2

## Overview

Added Devicon tech stack icon rows to experience cards and inline keyword highlighting to bullet points. Both features are data-driven — engineers edit resume.md YAML/markdown, no code changes needed.

## Phases

### Phase 5: Tech Stack Icons

**Goal**: Each experience card displays the role's tech stack at a glance via colored Devicon icons
**Depends on**: Nothing (additive UI layer over existing experience cards)
**Plans**: 1 plan

Plans:
- [x] 05-01: Add tech_stack type, CDN link, TechStackIcons component, wire into WorkExperience

**Details:**
- `ExperienceEntry` gains `tech_stack?: string[]` optional field in `src/types/resume.ts`
- `src/app/layout.tsx` loads Devicons CDN stylesheet via `<link>` in `<head>` — no npm package
- `TechStackIcons.tsx` Server Component with 30-entry SLUG_MAP allowlist; unknown techs → zinc pill `<span>`
- Renders between role/date header and bullets `<ul>` in `WorkExperience.tsx`
- Returns null for undefined/empty stack — zero layout shift on entries without tech_stack

### Phase 6: Keyword Highlights

**Goal**: Recruiters can immediately spot key technologies and terms in bullet points via accent-colored text
**Depends on**: Nothing (independent of Phase 5)
**Plans**: 1 plan

Plans:
- [x] 06-01: Create HighlightedBullet component, integrate into WorkExperience, add sample markup data

**Details:**
- `HighlightedBullet.tsx` Server Component — two-pass regex split: `**bold**` → indigo-600 spans, `*italic*` → italic spans
- `WorkExperience.tsx` wraps each bullet `<li>` with `<HighlightedBullet>{bullet}</HighlightedBullet>`
- Plain bullets → bare text nodes, no wrapper span — identical to prior behavior
- No npm packages — pure regex split with capturing group

## Milestone Summary

**Key Decisions:**
- Devicons loaded via CDN stylesheet link in layout.tsx head — no npm package added
- SLUG_MAP allowlist with 30 entries controls CSS class construction (security: T-05-02)
- Unknown tech names → zinc text pills, never injected into class attributes
- Bold parsed before italic to safely handle `***triple***` edge case
- Use span-only output (no strong/em/b/i) — color and italic are decorative, not semantic

**Issues Resolved:**
- Pre-existing `framer-motion`, `gray-matter`, and `babel-plugin-react-compiler` missing from node_modules — fixed with `npm install` during Phase 5 execution
- Biome formatter required semicolons — applied `biome format --write` to resume.ts

**Issues Deferred:**
- Human visual verification of CDN icon rendering, pill fallback legibility, layout regression without tech_stack — browser-only, non-blocking

**Technical Debt:**
- Phase 6 VALIDATION.md absent (non-blocking per audit)
- Phase 5 browser visual checks remain human-only (CDN rendering cannot be auto-verified)
