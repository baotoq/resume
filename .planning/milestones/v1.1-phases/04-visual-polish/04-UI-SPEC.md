## Spacing Scale

Declared values (multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Inline gaps within text, dot border width |
| sm | 8px | Gap between logo and company/role text |
| md | 16px | Card internal padding sides (px-6 = 24px per existing card — see exception) |
| lg | 24px | Card internal padding (existing `px-6 py-6`); gap between cards (`gap-6`) |
| xl | 32px | Page-level horizontal margin on mobile fallback |
| 2xl | 48px | — |
| 3xl | 64px | — |

Exceptions:
- Cards use `px-6 py-6` (24px) — inherited from existing component; do not change.
- Logo slot: 40×40px fixed — not on the 8-point scale but a locked visual decision (D-05).
- Timeline rail width: 16px (4×4) on desktop; collapses to 12px on mobile 375px to preserve card width.
- Touch target minimum: 44px for any interactive element (none in this phase — logos are not interactive).

Source: existing WorkExperience.tsx Tailwind classes; CONTEXT.md D-05, D-11

## Typography

Existing type scale — do not introduce new sizes.

| Role | Size | Weight | Line Height | Tailwind class |
|------|------|--------|-------------|----------------|
| Display (name) | 28px | 600 (semibold) | 1.1 | `text-[28px] font-semibold leading-[1.1]` |
| Heading (section, company, role) | 20px / 14px | 600 / 400 | 1.2 | `text-xl font-semibold` / `text-sm font-normal` |
| Body (bullets) | 16px | 400 (normal) | relaxed (~1.625) | `text-base leading-relaxed` |
| Label (dates, meta) | 14px | 400 (normal) | — | `text-sm font-normal` |

Phase 4 introduces no new type roles. Logo alt text is read by screen readers; company name text is the visible label (already rendered by existing card).

Source: existing WorkExperience.tsx and Header.tsx Tailwind classes

## Color

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `#fafafa` (zinc-50) | Page background (`bg-zinc-50`) |
| Secondary (30%) | `#ffffff` (white) | Card surfaces (`bg-white`), rail background |
| Accent (10%) | `#4f46e5` (indigo-600) | Filled dot for current role only; link text in Header |
| Destructive | not applicable | No destructive actions in this phase |

Accent reserved for:
- **Filled timeline dot** for the current role (endDate === null) — `bg-indigo-600`
- **Link text** in Header (pre-existing; not changed in this phase)

Past-role dots: hollow/outlined — `border-2 border-zinc-300 bg-white` (matches card border palette).

Timeline vertical line: `bg-zinc-200` — matches existing `border-zinc-200` card border.

Briefcase fallback SVG color: `text-zinc-400` / `fill-zinc-400` — muted, does not compete with company text.

Source: CONTEXT.md D-09, Claude's Discretion; globals.css; existing Tailwind classes throughout components

- Props: `{ src: string | undefined; alt: string }`

- onError + useState triggers briefcase SVG fallback

## Mobile Behavior (Claude's Discretion — D-11)

At 375px viewport:
- Rail width collapses from 28px left padding to 20px — rail line and dots remain visible
- Cards do not collapse or hide the timeline — the rail persists at all widths
- Logo remains 40×40px — it fits within the card header at 375px alongside company/role text (card has 24px horizontal padding on each side; available text width ≈ 375 - 48 - 20 - 40 - 12 ≈ 255px — sufficient for company/role text)
- No horizontal scroll introduced
- Date moves below company/role on small screens (`sm:flex-row` breakpoint already in existing card; preserve this pattern)

## Interaction Contract

| Interaction | Behavior |
|-------------|----------|
| Logo load success | `<img>` renders at 40×40px, `rounded-lg` |
| Logo load failure (onError) | State flips to fallback; briefcase SVG renders in same 40×40 slot; no layout shift |
| Logo absent (no logo_url) | LogoImage receives `src={undefined}`; renders briefcase immediately without attempting fetch |
| Timeline dot — current role | Filled indigo-600 circle, 12×12px |
| Timeline dot — past role | Hollow circle, `border-2 border-zinc-300 bg-white`, 12×12px |
| Vertical line | Static; no animation |
| Hover/focus | No new hover/focus states introduced — logos are not interactive links |

---

## Accessibility

- Logo `<img>` alt: `"{company} logo"` — descriptive, non-empty
- Briefcase fallback `<svg>` must include `aria-label="Company logo unavailable"` or `role="img"` with title
- Dot elements are decorative — `aria-hidden="true"` on dot and line markup
- Timeline line is decorative — `aria-hidden="true"`
- No new keyboard interactions introduced
