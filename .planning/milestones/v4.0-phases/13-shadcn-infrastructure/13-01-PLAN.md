CRITICAL values to use (UI-SPEC is authoritative for visual preservation):
- `--foreground: oklch(0.210 0.006 286)` — NOT `oklch(0.145 0 0)`. The PATTERNS.md merge target uses 0.145 which is shadcn's default, but it is significantly darker than the existing `#18181b` (zinc-900). The UI-SPEC specifies `oklch(0.210 0.006 286)` as the precise equivalent that preserves zero visual change. Use the UI-SPEC value.
- `--card-foreground: oklch(0.210 0.006 286)` — match foreground so card text has visual parity with body text. The shadcn default `oklch(0.145 0 0)` would make card text slightly darker than body text in Phase 13 (before Phase 14 swaps card elements), violating the zero-visual-change contract.
