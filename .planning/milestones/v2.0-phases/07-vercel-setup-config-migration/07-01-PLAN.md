**.github/workflows/deploy.yml — full replacement per D-06, D-07, D-08, D-09, D-10:**

Replace the entire file with the following YAML (do NOT amend the old file — replace it completely):

```yaml
name: deploy
env:
  VERCEL_ORG_ID: ${{ secrets.VERCEL_ORG_ID }}
  VERCEL_PROJECT_ID: ${{ secrets.VERCEL_PROJECT_ID }}
on:
  push:
    branches: ["master"]
  workflow_dispatch:
concurrency:
  group: "vercel-production"
  cancel-in-progress: false
jobs:
  deploy:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - uses: actions/setup-node@v4
        with:
          node-version: "20"
          cache: "npm"
      - run: npm ci
      - run: npm install --global vercel@latest
      - run: vercel pull --yes --environment=production --token=${{ secrets.VERCEL_TOKEN }}
      - run: vercel build --prod --token=${{ secrets.VERCEL_TOKEN }}
      - run: vercel deploy --prebuilt --prod --token=${{ secrets.VERCEL_TOKEN }}
```

Key differences from old file: No `permissions:` block (per D-09). Concurrency group is `"vercel-production"` (per D-10). Single `deploy` job (not build+deploy). Uses Vercel CLI three-step pattern (per D-07). Branch is `master` (per D-08).

NOTE: The VALIDATION.md references `vercel-deploy.yml` as a filename but per D-06 the correct approach is to replace `deploy.yml` in-place. This avoids Pitfall 5 (old workflow still running in parallel).

## STRIDE Threat Register

| T-07-02 | Tampering | MIME type sniffing | mitigate | `X-Content-Type-Options: nosniff` header in next.config.ts headers() — SEC-01 |
| T-07-03 | Info Disclosure | Referrer header leakage | mitigate | `Referrer-Policy: origin-when-cross-origin` in next.config.ts headers() — SEC-01 |
