## Visual Regression Constraint: Logo Image Rendering (IMG-01)

This is the only visual contract that changes in this phase.

**Before:** `LogoImage.tsx` renders logos via `next/image` with `images.unoptimized: true` — images pass through with no format conversion or resizing.

**After (IMG-01):** `images.unoptimized` is removed and `remotePatterns` is added. Next.js image optimization becomes active — Vercel's CDN will serve WebP, enforce size constraints, and lazy-load by default.

**Contract (must be verified post-deploy):**

| Property | Required Behavior |
|----------|------------------|
| Dimensions | Logo images must render at the same pixel dimensions as before migration — no layout shift |
| Aspect ratio | Aspect ratio must be preserved — no distortion |
| Loading | Lazy loading (Next.js default) is acceptable — above-fold logos may use `priority` prop if visible flash is observed |
| Format | WebP delivery is expected and acceptable |
| 404 prevention | All `logo_url` values in `src/data/resume.md` must resolve to actual image files (not HTML pages) — Clearbit Logo API (`https://logo.clearbit.com/<domain>`) is the recommended source |

**Clearbit hostname:** If Clearbit is used, add `{ protocol: "https", hostname: "logo.clearbit.com" }` to `remotePatterns` in `next.config.ts`.
