  <how-to-verify>
    Complete these 5 steps in order:

    1. **Create Vercel project:**
       ```bash
       npx vercel@latest link
       ```
       Follow prompts to create/link the project. When done, read the IDs:
       ```bash
       cat .vercel/project.json
       ```
       Note the `orgId` and `projectId` values.

    2. **Add GitHub repo secrets** (GitHub repo -> Settings -> Secrets and variables -> Actions -> New repository secret):
       - `VERCEL_TOKEN` = your Vercel API token (create at Vercel Dashboard -> Account Settings -> Tokens)
       - `VERCEL_ORG_ID` = the `orgId` from step 1
       - `VERCEL_PROJECT_ID` = the `projectId` from step 1

    3. **Disable Vercel Git Integration auto-deploy** (prevents double deploys):
       Vercel Dashboard -> Project -> Settings -> Git -> "Ignored Build Step" -> enter `exit 1` -> Save

    4. **Add environment variables in Vercel dashboard:**
       Vercel Dashboard -> Project -> Settings -> Environment Variables:
       - `EMAIL` = your email address (select "Production" environment)
       - `PHONE` = your phone number (select "Production" environment)
       Do NOT use the `NEXT_PUBLIC_` prefix.

    5. **Update .env.local if it exists locally:**
       If you have a `.env.local` file, rename `NEXT_PUBLIC_EMAIL` to `EMAIL` and `NEXT_PUBLIC_PHONE` to `PHONE`.

    Type "done" when all 5 steps are complete.
  </how-to-verify>
  <resume-signal>Type "done" when all steps are complete, or describe any issues encountered.</resume-signal>
  <acceptance_criteria>

    - .vercel/project.json exists locally (from vercel link)
    - Three GitHub secrets exist: VERCEL_TOKEN, VERCEL_ORG_ID, VERCEL_PROJECT_ID
    - Two Vercel env vars exist: EMAIL, PHONE
    - Vercel Git Integration auto-deploy is disabled (Ignored Build Step = exit 1)
