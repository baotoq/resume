Create `src/components/HighlightedBullet.tsx` as a React Server Component (NO `'use client'` directive).

Interface:
```typescript
interface HighlightedBulletProps {
  children: string;
}
```

Export: `export function HighlightedBullet({ children }: HighlightedBulletProps)`

Parsing logic — pure regex, no npm packages (per D-04):

1. FIRST pass — split on bold `**...**` using regex `/\*\*([^*]+)\*\*/g`:
   - Use `String.prototype.split()` with a capturing group regex to get alternating [text, bold, text, bold, ...] segments
   - Bold segments (odd indices from split) render as `<span className="text-indigo-600">{text}</span>` — color only, NO font-weight change, NO `<strong>` tag (per D-01)

2. SECOND pass — within each non-bold segment, split on italic `*...*` using regex `/\*([^*]+)\*/g`:
   - Italic segments render as `<span className="italic">{text}</span>` — NO `<em>` tag
   - Plain text segments render as bare React text nodes

3. `***triple***` handling: because bold is parsed first, `***text***` will be split as `*` + bold(`text`) + `*` — the stray `*` characters pass through as literal text. This is the safe fallback per CONTEXT.md discretion.

Return value: `<>{segments}</>` — fragment only, NO wrapper element. The component renders inline inside a `<li>`.

Key rules:
- NO `'use client'` — this is a Server Component
- NO `useMemo`, `useCallback`, or any hooks — React Compiler handles optimization
- NO `<strong>`, `<b>`, `<em>`, `<i>` semantic tags — use `<span>` only (per UI-SPEC accessibility contract)
- Each `<span>` must have a `key` prop (use index-based keys since this is static content)
