## Directory Purposes

**`src/app/`:**
- Purpose: Next.js App Router route segments — each `page.tsx` is a publicly accessible route
- Contains: Root layout, home page, global CSS, favicon
- Key files: `src/app/layout.tsx`, `src/app/page.tsx`, `src/app/globals.css`

**`src/components/`:**
- Purpose: Reusable UI components used by pages
- Contains: Server Components (default) and Client Components (marked `"use client"`)
- Key files: `Header.tsx`, `WorkExperience.tsx`, `Skills.tsx`, `AnimateIn.tsx`, `LogoImage.tsx`

**`src/data/`:**
- Purpose: Static content files — the resume's single source of truth
- Contains: `resume.md` with YAML frontmatter only (no Markdown body content used)
- Key files: `src/data/resume.md` — edit this file to change all displayed resume content

**`src/types/`:**
- Purpose: Manually authored TypeScript interfaces for application data shapes
- Contains: `ResumeData` and `ExperienceEntry` interfaces
- Key files: `src/types/resume.ts`

**`types/`:**
- Purpose: Next.js auto-generated type declarations — regenerated on `next build` / `next dev`
- Generated: Yes — do not edit manually

**`public/`:**
- Purpose: Static assets accessible directly at the root URL (e.g., `/globe.svg`)
- Generated: No — commit files here intentionally

**`out/`:**
- Purpose: Static export output produced by `npm run build`
- Generated: Yes — do not commit (in `.gitignore`)

## Key File Locations

**Entry Points:**
- `src/app/layout.tsx`: Root layout — rendered for every route; sets fonts, metadata, global styles
- `src/app/page.tsx`: Home page at `/`; reads resume data and composes all sections

**Resume Content:**
- `src/data/resume.md`: YAML frontmatter — the only file to edit when updating resume content

**Types:**
- `src/types/resume.ts`: `ResumeData` and `ExperienceEntry` interfaces — update when adding new data fields

**Configuration:**
- `next.config.ts`: Static export mode, basePath `/resume`, React Compiler flag
- `tsconfig.json`: Path alias `@/*` → `src/*`, strict mode
- `biome.json`: Linting and formatting rules
- `postcss.config.mjs`: Tailwind v4 PostCSS integration

**Styles:**
- `src/app/globals.css`: Tailwind v4 import, CSS custom properties, `@theme inline` token bindings

## Naming Conventions

**Files:**
- React components: PascalCase matching export name (e.g., `WorkExperience.tsx` exports `WorkExperience`)
- Route files: lowercase required by Next.js (`page.tsx`, `layout.tsx`, `error.tsx`, `loading.tsx`)
- Utilities/helpers: camelCase (e.g., `formatDate.ts`)
- Data files: kebab-case (e.g., `resume.md`)

**Directories:**
- Route segments: kebab-case (e.g., `src/app/work-experience/`)
- Route groups (no URL impact): parentheses `(group)/`
- Feature folders under `src/`: lowercase plural nouns (`components/`, `data/`, `types/`)

**Exports:**
- Named exports used for all components (e.g., `export function Header`) — no default exports in components
- Default export required only for Next.js route files (`page.tsx`, `layout.tsx`)

## Where to Add New Code

**New resume section:**
- Add data fields to `src/types/resume.ts`
- Add corresponding YAML fields to `src/data/resume.md`
- Create `src/components/NewSection.tsx` as a Server Component
- Import and render in `src/app/page.tsx`, wrapped in `<AnimateIn>`

**New page route:**
- Create `src/app/[route-name]/page.tsx` with a default export component
- Optionally add `src/app/[route-name]/layout.tsx` for route-specific shell

**New shared component:**
- Server Component (default): `src/components/ComponentName.tsx` — no directive needed
- Client Component (needs state/browser APIs): `src/components/ComponentName.tsx` with `"use client"` as first line

**New utility/helper:**
- Create `src/lib/` directory and add `src/lib/helperName.ts`
- Import using path alias: `import { foo } from "@/lib/foo"`

**New static asset:**
- Place in `public/` — accessible at `/<filename>`
- Reference in JSX as a plain `<img src="/filename.svg">` or via `next/image`

**Path alias:**
- Always use `@/` prefix for imports from `src/`: `import { Header } from "@/components/Header"`

## Special Directories

- Generated: No
- Committed: Yes

**`types/`:**

- Purpose: Static HTML/CSS/JS export for deployment
- Generated: Yes
- Committed: No

**`.next/`:**
- Purpose: Next.js dev/build cache
- Generated: Yes
- Committed: No
